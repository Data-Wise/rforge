    SPEC: r:rhub Command (Unified)¶

    Date: 2026-06-20 Status: READY — supersedes both prior rhub specs; all gaps closed by adversarial review Supersedes: - SPEC-rhub-platform-selection-2026-06-19.md (platform selection + broken-platform guard) - SPEC-rhub-pak-stable-guard-2026-06-20.md (pak-version pre-flight check) Scope: lib/rcmd.py + commands/r/rhub.md (update existing) + tests/test_rcmd_rhub.py (new) Priority: High — existing r:rhub command is broken (hangs headlessly)

    Problem Statement¶

    rforge's existing r:rhub command (commands/r/rhub.md, lib/rcmd.py) is non-functional:

    1. Hangs headlessly — rhub::rhub_check(path) without platforms= arg opens an interactive console menu; _invoke_r() has no TTY → hangs permanently.
    2. rhub_setup() on every run — calls rhub::rhub_setup() unconditionally; this makes a git commit every invocation even when .github/workflows/rhub.yaml already exists.
    3. Broken platform drift — macos runner (macos-13) retired December 2025; any dispatch selecting it fails immediately.
    4. Silent pak failures — r-hub/actions/setup-deps@v1 defaults to pak devel (0.10.0.9000), which has a bootstrap regression (r-lib/pak #887); all jobs fail silently.
    5. Stale winbuilder duplicate — r_snippet("winbuilder", platform="rhub") in rcmd.py (lines 497–501) calls rhub::rhub_check(path) headlessly with no platforms arg — same hang bug, different entry point.

    User Decisions (from 2026-06-19 Q&A session)¶

    Q Decision
    Q1. Command structure Dedicated r:rhub command — update existing commands/r/rhub.md, not r:winbuilder
    Q2. Platform UX Never pass NULL headlessly — default to cran-submission preset when no platforms given
    Q3. Broken macos platform Hard-block + suggest macos-arm64
    Q4. Runner mode Both modes — rhub_check() (own GH account) + rc_submit() (RC shared) via --rc-mode
    Q5. Result surfacing Print URL + open in browser — construct from git remote, not stdout parse
    Q6. CRAN preset OS matrix + atlas: linux, windows, macos-arm64, atlas
    Q7. Setup check Hard-stop if .github/workflows/rhub.yaml missing — emit rhub_yaml_missing finding
    Q8. Sanitizers clang-asan in strict preset; atlas in default preset; mark clang-asan [unstable]

    rhub.yaml Default Config Advisory¶

    The default rhub.yaml generated by rhub_setup() has: 

    config:
  default: 'linux,windows,macos'
    macos in this default is a broken platform. When dispatching, rforge uses the platforms= argument to rhub_check() explicitly — it never relies on rhub's default config field. However, if the default config field contains macos, emit an advisory (rhub_yaml_default_broken_platform) recommending the user update the field to 'linux,windows,macos-arm64'. This is advisory-only (doesn't block dispatch).

    Architecture: Pre-flight Sequence¶

    All checks run before any R dispatch. Sequence is fixed:

    _rhub_preflight(pkg_path, platforms)
  │
  ├── (a) yaml_missing check
  │     rhub.yaml absent? → severity: 'error', finding: rhub_yaml_missing
  │     HARD-STOP: return error envelope immediately
  │
  ├── (b) broken_platform check
  │     any platform in _RHUB_BROKEN_PLATFORMS? → severity: 'error', finding: rhub_broken_platform
  │     HARD-STOP: return error envelope immediately
  │
  └── (c) pak_version check (_check_rhub_yaml)
        any setup-deps block missing pak-version: stable?
        → severity: 'advisory', finding: rhub_pak_devel_regression
        → DOES NOT block dispatch
        │
        (d) rhub_yaml_default_broken_platform check
        default config field contains broken platform?
        → severity: 'advisory', finding: rhub_yaml_default_broken_platform
        → DOES NOT block dispatch

    After preflight: - If any error findings → return error envelope, do NOT dispatch - If only advisory findings → dispatch normally, include findings in returned envelope - If no findings → dispatch normally

    Critical invariant: advisory findings NEVER short-circuit dispatch. The early-return pattern in SPEC-rhub-pak-stable-guard (if findings: return {...}) is incorrect and must NOT be replicated.

    1b. _check_rhub_yaml(pkg_path) helper¶

    Scans .github/workflows/rhub.yaml for setup-deps blocks missing pak-version: stable AND for broken platform names in the default config field.

    def _check_rhub_yaml(pkg_path: str) -> list[dict]:
    # TODO: remove or loosen when r-lib/pak #887 is fixed in devel
    # Upstream: https://github.com/r-lib/pak/issues/887
    import re
    yaml_path = Path(pkg_path) / ".github" / "workflows" / "rhub.yaml"
    if not yaml_path.exists():
        return []  # rhub_yaml_missing is handled by _rhub_preflight

    content = yaml_path.read_text()
    findings = []

    # Check pak-version in every setup-deps block
    blocks = re.split(r'- uses: r-hub/actions/setup-deps', content)
    # blocks[0] = content before first occurrence; blocks[1:] = after each
    for i, block in enumerate(blocks[1:], 1):
        with_match = re.search(r'\s+with:\s*((?:\s+\S.*\n)*)', block)
        if not with_match:
            continue
        with_block = with_match.group(1)
        if "pak-version:" not in with_block:
            findings.append({
                "code": "rhub_pak_devel_regression",
                "severity": "advisory",
                "block": i,   # informational: which setup-deps block (1-indexed)
                "message": (
                    f"setup-deps block {i} is missing `pak-version: stable`. "
                    "pak devel (0.10.0.9000) has a bootstrap regression "
                    "(r-lib/pak #887, filed 2026-06-13) — rhub jobs will "
                    "silently fail. Add `pak-version: stable` as the first "
                    "`with:` entry in this block."
                ),
                "fix": (
                    "- uses: r-hub/actions/setup-deps@v1\n"
                    "  with:\n"
                    "    pak-version: stable\n"
                    "    token: ${{ secrets.RHUB_TOKEN }}\n"
                    "    job-config: ${{ matrix.config.job-config }}"
                ),
            })

    # Check default config field for broken platforms
    default_match = re.search(r"default:\s*'([^']+)'", content)
    if default_match:
        default_platforms = [p.strip() for p in default_match.group(1).split(",")]
        broken_in_default = [p for p in default_platforms if p in _RHUB_BROKEN_PLATFORMS]
        if broken_in_default:
            findings.append({
                "code": "rhub_yaml_default_broken_platform",
                "severity": "advisory",
                "platforms": broken_in_default,
                "message": (
                    f"rhub.yaml default config includes retired platform(s): "
                    f"{', '.join(broken_in_default)}. Update the default field to "
                    f"'linux,windows,macos-arm64' to avoid confusing others who "
                    f"manually trigger the workflow."
                ),
            })

    return findings

    1c. _rhub_preflight(pkg_path, platforms) unified pre-flight¶

    def _rhub_preflight(pkg_path: str, platforms: list) -> list[dict]:
    """Unified pre-flight checks for r:rhub. Returns list of findings."""
    findings = []
    yaml_path = Path(pkg_path) / ".github" / "workflows" / "rhub.yaml"

    # (a) yaml_missing — hard error
    if not yaml_path.exists():
        findings.append({
            "code": "rhub_yaml_missing",
            "severity": "error",
            "message": (
                ".github/workflows/rhub.yaml not found. "
                "Run `rhub::rhub_setup()` in R to create it (requires a GitHub remote). "
                "This file must exist and be committed before r:rhub can dispatch."
            ),
        })
        return findings  # No further checks possible

    # (b) broken_platform — hard error per broken platform
    for plat in platforms:
        if plat in _RHUB_BROKEN_PLATFORMS:
            findings.append({
                "code": "rhub_broken_platform",
                "severity": "error",
                "platform": plat,
                "message": (
                    f"Platform '{plat}' is broken: {_RHUB_BROKEN_PLATFORMS[plat]}"
                ),
            })

    # (c) pak_version + default config advisories
    findings.extend(_check_rhub_yaml(pkg_path))

    return findings

    1d. URL construction helper¶

    def _rhub_actions_url(pkg_path: str) -> str:
    """Construct GitHub Actions URL from git remote. Returns empty string on failure."""
    import subprocess
    try:
        result = subprocess.run(
            ["git", "-C", pkg_path, "remote", "get-url", "origin"],
            capture_output=True, text=True, timeout=5
        )
        remote = result.stdout.strip()
        # Normalize: ssh -> https, strip .git suffix
        if remote.startswith("git@github.com:"):
            remote = "https://github.com/" + remote[len("git@github.com:"):]
        if remote.endswith(".git"):
            remote = remote[:-4]
        return remote.rstrip("/") + "/actions"
    except Exception:
        return ""

    1f. run() wire-in for kind="rhub"¶

    # In run(), when kind == "rhub":
platforms = kwargs.get("platforms", None)  # list[str] or None
preset = kwargs.get("preset", None)        # str or None
rc_mode = kwargs.get("rc_mode", False)     # bool

# Resolve platforms: preset → explicit list → default preset
if preset is not None:
    if preset not in _RHUB_PRESETS:
        return {
            "kind": "rhub", "status": "error", "engine_missing": [],
            "messages": [
                f"Unknown preset '{preset}'. "
                f"Valid presets: {', '.join(_RHUB_PRESETS)}"
            ],
        }
    platforms = _RHUB_PRESETS[preset]
elif platforms is None:
    # Never pass NULL headlessly — fall back to default preset
    platforms = _RHUB_PRESETS["cran-submission"]

# Pre-flight
preflight = _rhub_preflight(path, platforms)
errors = [f for f in preflight if f.get("severity") == "error"]
if errors:
    return {
        "kind": "rhub", "status": "error", "engine_missing": [],
        "messages": [f["message"] for f in errors],
    }
advisories = [f for f in preflight if f.get("severity") == "advisory"]

# Dispatch
env = _invoke_r(r_snippet("rhub", path, platforms=platforms, rc_mode=rc_mode))
if env.get("status") == "error":
    return env

# Construct and open URL
actions_url = _rhub_actions_url(path)
if actions_url:
    import webbrowser
    webbrowser.open(actions_url)

# Merge advisory findings into envelope
env["findings"] = advisories
env["run_url"] = actions_url or None
return env

    Non-Goals¶

    • Polling GitHub Actions for job status (needs GH token, out of scope)
    • Auto-patching rhub.yaml (too invasive — user owns workflow files)

    • Auto-configuring PAT or GitHub remote
    • Running rhub_setup() automatically (user must run it in R; rforge detects the file)
