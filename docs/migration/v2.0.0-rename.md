# Migrating to rforge v2.0.0

> **3 commands renamed.** Update your habits or scripts using the table below.
> The other 13 commands are unchanged.

## Rename mapping

| You used to type            | In v2.0.0+, type        | Why |
|-----------------------------|-------------------------|-----|
| `/rforge:doc-check`         | `/rforge:docs:check`    | Aligns with craft's `docs:` namespace |
| `/rforge:ecosystem-health`  | `/rforge:health`        | Shorter, daily-use friendly           |
| `/rforge:rpkg-check`        | `/rforge:r:check`       | R-specific commands get an `r:` prefix |

Unchanged: `analyze`, `capture`, `cascade`, `complete`, `deps`, `detect`,
`impact`, `init`, `next`, `quick`, `release`, `status`, `thorough`.

## Safety net: typing the old name still works (briefly)

Each renamed command keeps a stub at its old name that responds with a
rename-error pointing at the new name. So typing `/rforge:doc-check`
in Claude Code produces:

> ❌ **Renamed:** this command (`/rforge:doc-check`) was renamed in v2.0.0.
>
> **Use instead:** `/rforge:docs:check`
>
> See the migration tutorial: <https://data-wise.github.io/rforge/migration/v2.0.0-rename/>

You won't silently get nothing — you'll get a helpful pointer. The stubs
are retained through the v2.x line and slated for removal in v3.0.0.

## Why these renames?

rforge's namespacing now mirrors craft's **hybrid** scheme:

- **High-frequency daily commands stay flat** (`/rforge:status`,
  `/rforge:next`, `/rforge:analyze`). Typing a colon-namespace prefix for
  a command you run 20× a day adds friction without payoff.
- **Sub-namespaces are used only where there's a real grouping.** craft
  has a `docs:` sub-namespace for documentation operations, so rforge's
  documentation drift check moves to `docs:check` for cross-plugin
  consistency. R-specific commands get their own `r:` prefix.
- **Single-command sub-namespaces are avoided.** `ecosystem-health` was
  a one-off — wrapping it in a `health:` sub-namespace would force you
  to type more for no grouping benefit, so it becomes the shorter,
  flat `health`.

The net is a **smaller** breaking change than a full namespacing
overhaul, hitting only the 3 commands where the old names felt either
out of step with craft (`doc-check` vs craft's `docs:*`) or
unnecessarily long (`ecosystem-health`, `rpkg-check`).

For the longer rationale and rejected alternatives, see the SPEC at
[`docs/specs/SPEC-phase3-namespacing-2026-05-11.md`](../specs/SPEC-phase3-namespacing-2026-05-11.md).

## What about my scripts?

If you have shell scripts, aliases, or notes referencing the old names,
update them in two steps.

### 1. Find every reference

```bash
grep -rn -E '/rforge:(doc-check|ecosystem-health|rpkg-check)' \
    ~/your-project ~/your-notes
```

### 2. Mass-update in place

POSIX `sed` (works on both macOS and Linux):

```bash
# Preview the changes first (no -i)
find . -type f \( -name '*.sh' -o -name '*.md' -o -name '*.txt' \) \
  -exec sed -E \
    -e 's|/rforge:doc-check|/rforge:docs:check|g' \
    -e 's|/rforge:ecosystem-health|/rforge:health|g' \
    -e 's|/rforge:rpkg-check|/rforge:r:check|g' \
    {} \;

# Apply in place (after preview looks right)
# macOS:
find . -type f \( -name '*.sh' -o -name '*.md' -o -name '*.txt' \) \
  -exec sed -i '' -E \
    -e 's|/rforge:doc-check|/rforge:docs:check|g' \
    -e 's|/rforge:ecosystem-health|/rforge:health|g' \
    -e 's|/rforge:rpkg-check|/rforge:r:check|g' \
    {} \;

# Linux (GNU sed):
find . -type f \( -name '*.sh' -o -name '*.md' -o -name '*.txt' \) \
  -exec sed -i -E \
    -e 's|/rforge:doc-check|/rforge:docs:check|g' \
    -e 's|/rforge:ecosystem-health|/rforge:health|g' \
    -e 's|/rforge:rpkg-check|/rforge:r:check|g' \
    {} \;
```

The `|` separator avoids quoting headaches with the `:` in the new names.

## Anything else changed?

No. v2.0.0's only behavioral break is the 3 renames. APIs, config
options, hook contracts, lib/* modules, and state files are unchanged
from v1.3.0.
