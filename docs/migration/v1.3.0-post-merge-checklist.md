# v1.3.0 post-merge checklist (cross-repo work)

After the rforge plugin v1.3.0 PR merges to `dev`/`main`, complete these
out-of-tree steps. **None of them touch the rforge repo itself** — they all
operate on the (now-obsolete) `data-wise/rforge-mcp` repo.

> **Why post-merge?** Archiving rforge-mcp before v1.3.0 lands would leave
> the replacement under review while the original is read-only — bad UX
> for anyone watching the org. Wait for the merge, then run these steps.

## 1. Push DEPRECATED.md to rforge-mcp

```bash
# Adjust this path to wherever you have rforge-mcp cloned locally.
# Common locations: ~/projects/data-wise/rforge-mcp or ~/code/rforge-mcp
cd ~/projects/data-wise/rforge-mcp

git checkout main
git pull

# Copy the drafted text from the merged rforge PR into ./DEPRECATED.md
# (adjust the source path if your rforge clone lives elsewhere)
cp ~/projects/dev-tools/rforge/docs/migration/rforge-mcp-deprecation.md ./DEPRECATED.md

# Edit DEPRECATED.md: replace [DATE TO BE FILLED IN AT ARCHIVE TIME]
# with today's date (YYYY-MM-DD).
$EDITOR DEPRECATED.md

git add DEPRECATED.md
git commit -m "docs: deprecation notice — absorbed into rforge plugin v1.3.0"
git push origin main
```

## 2. Archive the GitHub repo

```bash
gh repo archive data-wise/rforge-mcp --yes
```

This makes the repo read-only. Issues and PRs are blocked. The repo
remains visible and clonable. Reversible via the GitHub UI if needed
(Settings → Danger Zone → Unarchive).

## 3. (Optional) Note to existing local users

If you have shell aliases, npm-linked binaries, or shortcuts pointing at
a local `rforge-mcp` install, remove them. The plugin now provides
everything via `python3 -m lib.<tool>` and the `/rforge:*` slash commands.

## 4. Update your own settings.json

If your personal `~/.claude/settings.json` has an `mcpServers.rforge`
entry, remove it manually. We don't auto-edit user settings. Example:

```json
// Before:
"mcpServers": {
  "rforge": { "command": "...", "args": [...] },
  "other-server": { ... }
}

// After:
"mcpServers": {
  "other-server": { ... }
}
```

## Verification

After completing steps 1 and 2:

- `gh repo view data-wise/rforge-mcp` should show the archived banner.
- Visiting https://github.com/data-wise/rforge-mcp should display the
  "This repository has been archived..." notice at the top.
- `gh api repos/data-wise/rforge-mcp --jq .archived` should return `true`.
- The `DEPRECATED.md` file should be visible at the root of the rforge-mcp
  repo on the `main` branch.

## What's NOT needed

No npm, Homebrew, or PyPI cleanup is required — rforge-mcp was never
published to any package registry. The only artifacts are the GitHub repo
itself and any local clones / `npm link` installs on individual machines.

## Rollback

If something goes wrong:

- **DEPRECATED.md commit**: `git revert` the commit on rforge-mcp `main`.
- **Archive**: GitHub UI → repo Settings → Danger Zone → Unarchive
  repository. The repo becomes writable again immediately.
